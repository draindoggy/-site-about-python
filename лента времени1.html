История языка программирования Python началась в конце 1980-х. Гвидо ван Россум задумал Python в 1980-х годах, а приступил к его созданию в декабре 1989 года в центре математики и информатики в Нидерландах. Ван Россум является основным автором Python и продолжал выполнять центральную роль в принятии решений относительно развития языка вплоть до 12 июля 2018 года.

Ранняя история

В феврале 1991 года Ван Россум опубликовал код (обозначенный как версия 0.9.0) в alt.sources. На этом этапе разработки уже присутствовали классы с наслеованием, обработкой исключений, функциями и основными типми listданных , dict, strи так далее. Также в этом первоначальном выпуске была модульная система, заимствованная из Modula-3; Ван Россум описывает модуль как "одну из основных программных единиц Python".Модель исключений Python также напоминает Modula-3, с добавлением else предложения. В 1994 году был создан comp.lang.python, основной дискуссионный форум по Python, что стало важной вехой в развитии пользовательской базы Python.

  • Версия 1

    • Python достиг версии 1.0 в январе 1994 года. Основными новыми функциями, включенными в этот выпуск, были инструменты функционального программирования lambda, map, filterи reduce. Ван Россум заявил, что "Python приобрел lambda, reduce(), filter() и map(), любезно предоставленные хакером Lisp, который пропустил их и отправил рабочие исправления". К версии 1.4 Python приобрел несколько новых функций. Среди них следует отметить аргументы ключевых слов, основанные на Modula-3 (которые также похожи на аргументы ключевых слов Common Lisp) и встроенную поддержку комплексных чисел. Во время пребывания Ван Россума в CNRI он запустил инициативу Computer Programming for Everybody (CP4E), намереваясь сделать программирование более доступным для большего числа людей с базовой "грамотностью" в языках программирования, аналогичной базовой грамотности в английском и математических навыках, требуемых большинством работодателей. Python сыграл в этом центральную роль: из-за его ориентации на чистый синтаксис он уже был подходящим, а цели CP4E имели сходство с его предшественником ABC.

  • Версия 2

    • Python 2.0, выпущенный в октябре 2000 года, представил понимание списков, функцию, заимствованную из языков функционального программирования SETL и Haskell. Синтаксис Python для этой конструкции очень похож на синтаксис Haskell, за исключением предпочтений Haskell для знаков препинания и предпочтений Python для алфавитных ключевых слов. В Python 2.0 также был представлен сборщик мусора, способный собирать ссылочные циклы. Python 2.1 был близок к Python 1.6.1, а также к Python 2.0. Его лицензия была переименована в Python Software Foundation License. Весь код, документация и спецификации, добавленные со времени альфа-версии Python 2.1, принадлежат Python Software Foundation (PSF), некоммерческой организации, созданной в 2001 году по образцу Apache Software Foundation. Python 2.5 был выпущен в сентябре 2006 года и представил withинструкцию, которая помещает блок кода в контекстный менеджер (например, получение блокировки перед запуском блока кода и последующее снятие блокировки или открытие файла, а затем его закрытие), что позволяет инициализировать сбор ресурсов (Поведение, подобное RAII, и замена обычной идиомы try / finally. Python 2.6 был выпущен одновременно с Python 3.0 и включал некоторые функции из этого выпуска, а также режим "предупреждения", который подчеркивал использование функций, которые были удалены в Python 3.0. Аналогичным образом, Python 2.7 совпал с Python 3.1 и включал в себя функции из него, который был выпущен 26 июня 2009 года. В ноябре 2014 года было объявлено, что Python 2.7 будет поддерживаться до 2020 года, но пользователям было рекомендовано как можно скорее перейти на Python 3. Поддержка Python 2.7 прекратилась 1 января 2020 года вместе с замораживанием кода ветки разработки 2.7. Финальный релиз, 2.7.18, вышел 20 апреля 2020 года и включал исправления критических ошибок и блокировщиков релизов. Это ознаменовало конец существования Python 2.

  • Версия 3

    • Python 3.0 (также называемый "Python 3000" или "Py3K") был выпущен 3 декабря 2008 года.Он был разработан для исправления фундаментальных недостатков языка – требуемые изменения не могли быть реализованы при сохранении полной обратной совместимости с серией 2.x, что потребовало нового основного номера версии. Руководящим принципом Python 3 было: "уменьшите дублирование функций, удалив старые способы выполнения задач". Python 3.0 был разработан с той же философией, что и в предыдущих версиях. Однако, поскольку Python накопил новые и избыточные способы программирования одной и той же задачи, в Python 3.0 был сделан упор на удаление дублирующих конструкций и модулей, в соответствии с Дзен Python: "Должен быть один — и желательно только один — очевидный способ сделать это". Тем не менее, Python 3.0 оставался многопарадигмальным языком. Программисты все еще могли следовать, среди прочего, парадигмам объектно-ориентированного, структурированного и функционального программирования, но в рамках такого широкого выбора детали должны были быть более очевидными в Python 3.0, чем в Python 2.x.
